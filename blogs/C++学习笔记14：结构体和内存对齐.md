# C++学习笔记14：结构体和内存对齐

## 一、为什么要学习内存对齐？

在C/C++中，计算结构体或类对象大小时，不能只把所有成员变量的大小简单相加。

例如：

```cpp
struct A {
    char c;
    int i;
};
```

很多初学者可能认为：

```cpp
char 占 1 个字节
int 占 4 个字节
所以 A 的大小是 1 + 4 = 5 个字节
```

但实际使用 sizeof(A) 时，结果通常是：

```cpp
8
```


原因就是：**内存对齐**。

内存对齐是编译器为了提高内存的访问效率而采用的一种存储规则。

------

## 二、内存对齐规则

结构体和类的内存对齐规则基本相同。

简单记住下面几条：

1. 第一个成员放在偏移量为 0 的位置；
2. 每个成员要放在自己对齐数的整数倍位置；
3. 对齐数通常取“成员自身大小”和“编译器默认对齐数”中的较小值；
4. 整个结构体和类的大小，必须最大对齐数的整数倍。

很多编译器环境下，默认对齐数通常是 8。

------

## 三、例子一：char + int

代码：

```cpp
#include <iostream>
using namespace std;

struct A {
	char c;
	int i; 
}; 

int main() {
	cout << sizeof(A) << endl;
	return 0;
}
```

分析：

```cpp
char c 占 1 字节，放在偏移量为 0 的位置；
int i 占 4 个字节，需要放在 4 的整数倍位置；
偏移量 1、2、3 不能直接放 int，所以中间补 3 个字节；
int i 放在偏移量 4~7；
最终总大小是 8 字节。
```


所以通常结果是：

```cpp
8
```


内存大致可以理解为：

```cpp
c _ _ _ i i i i
```


其中 _ 表示填充字节。

------

## 四、例子二：int + char

在看一个成员顺序不同的例子：

```cpp
#include <iostream>
using namespace std;

struct B {
	int i;
    char c; 
}; 

int main() {
	cout << sizeof(B) << endl;
	return 0;
}
```


分析：

```cpp
int i 占 4 字节，放在偏移量 0~3；
char c 占 1 字节，放在偏移量 4；
最大对齐数是 4；
总大小必须是 4 的整数倍；
所以末尾补 3 个字节；
最终大小是 8 字节。
```


所以结果通常是：

```cpp
8
```


内存大致可以理解为：

```cpp
i i i i c _ _ _
```

------

## 五、例子三：char + int + char

再看一个稍微复杂一点的：

```cpp
#include <iostream>
using namespace std;

struct C {
	char c1;
	int i; 
	char c2;
}; 

int main() {
	cout << sizeof(C) << endl;
	return 0;
}
```


分析：

```cpp
char c1 占 1 个字节，放在偏移量 0；
int i 占 4 字节，需要放在 4 的整数倍位置；
所以偏移量 1、2、3 补齐；
int i 放在偏移量 4~7；
char c2 占 1 个字节，放在偏移量8；
最大对齐数是 4；
所以要是 4 的整数倍；
所以末尾补 3 个字节；
最终大小是 12 字节。
```


通常结果是：

```cpp
12
```

内存大致可以理解为：

```cpp
c1 _ _ _ i i i i c2 _ _ _
```

## 六、成员顺序可能影响大小

有时候，调整成员变量的顺序，可能减少内存的浪费。

例如：

```cpp
struct C {
    char c1;
    int i;
    char c2;
};
```

通常大小是 12。

如果改成：

```cpp
struct D {
    int i;
    char c1;
    char c2;
};
```


分析：

```cpp
int i 占 4 字节，放在偏移量 0~3；
char c1 占 1 字节，放在偏移量 4；
char c2 占 1 字节，放在偏移量 5；
当前使用 6 字节；
最大对齐数是 4；
总大小要补齐到 4 的整数倍；
最终大小是 8 字节。
```


所以 sizeof(D) 通常是：

```cpp
8
```


可以看到同样的成员变量，不同的排列顺序，最终大小可能不同。

因此在实际开发中，可以把占内存较大的成员变量放在前面，把较小的成员变量放后面，这样有时候可以减少内存的浪费。

------

## 七、类也遵守内存对齐规则

上面的例子用的是 struct，如果换成 class，内存对齐规则也是一样的。

例如：

```cpp
class E {
    private:
        char c;
        int i;
};
```


这个类的对象大小通常也是 8 字节。

struct 和 class 的主要区别是默认访问权限的不同：

```cpp
struct 默认是 public
class 默认是 private
```

他们的成员变量对齐规则是一样的。

------

## 八、做 sizeof 题的步骤

以后遇到结构体或类的 sizeof 题，可以按照这个步骤分析：

```cpp
第一步：确定每个成员变量的大小；
第二步：确定每个成员变量的对齐数；
第三步：按成员变量依次排偏移量；
第四步：中间不满足对齐要求时补字节；
第五步：最后总的大小补齐到最大对齐数的整数倍。
```


比如：

```cpp
struct A {
    char c;
    int i;
};
```


就按：

```cpp
char 放 0
int 要放 4 的倍数位置
中间补 3 个字节
总大小 8
```


这样分析就不容易错。

------

## 九、小结

本篇主要学习了结构体和类的内存对齐。

需要记住：

1. 结构体和类的大小不能简单相加；
2. 内存对齐是为了提高访问效率；
3. 第一个成员从偏移量 0 开始存放；
4. 每个成员要放到自己对齐数的整数倍；
5. 总的大小要是最大对齐数的整数倍；
6. 成员变量顺序可能影响最终大小；
7. struct 和 class 的内存对齐规则相同；
8. 做 sizeof 题时，要一步步分析偏移量。

内存对齐是 C/C++ 中很常见的考点，尤其是结构体和类对象大小的计算，需要多做几个例子才能熟练掌握。
