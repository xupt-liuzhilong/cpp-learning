# C++学习笔记18：拷贝构造与深浅拷贝

## 一、什么是拷贝构造函数？

拷贝构造函数是构造函数的一种特殊形式。

它的作用是：

> 用一个已经存在的同类型对象，去初始化一个新对象。

例如：

```cpp
Date d1(2024, 5, 1);
Date d2(d1);
```


这里 `d2` 是一个新对象，它使用已经存在的 `d1` 进行初始化，所以会调用拷贝构造函数。

------

## 二、拷贝构造函数的写法

拷贝构造函数的基本格式是：

```cpp
类名(const 类名& 对象名)
```

例如一个日期类的拷贝构造函数：

```cpp
#include <iostream>
using namespace std;

class Date {
	public:
	    Date(int year = 2024, int month = 1, int day = 1) {
	        _year = year;
	        _month = month;
	        _day = day;
	    }
	    
        // 拷贝构造函数
	    Date(const Date& d) {
	        _year = d._year;
	        _month = d._month;
	        _day = d._day;
	    }
	
	    void Print() {
	        cout << _year << "-" << _month << "-" << _day << endl;
	    }
	
	private:
	    int _year;
	    int _month;
	    int _day;
};

int main() {
    Date d1(2024, 5, 1);
    Date d2(d1);

    d1.Print();
    d2.Print();

    return 0;
}
```

运行结果：

```cpp
2024-5-1
2024-5-1
```

这里 d2 是通过 d1 初始化出来的，所以 d2 的年月日和 d1 一样。

------

## 三、什么时候调用拷贝构造函数？

拷贝构造函数常见的调用方式有两种：

### 1.用已有的对象初始化新对象

```cpp
Date d1(2024, 5, 1);
Date d2(d1);
```

这里 d2 是新创建出来的对象，所以调用拷贝构造函数。

------

### 2.使用 Date d2 = d1

```cpp
Date d1(2024, 5, 1);
Date d2 = d1;
```

这也是拷贝构造，不是赋值运算符。

原因是：

> d2 此时还没有存在，正在被创建，所以是初始化，不是赋值。

这一点很容易混淆，判断标准：

```cpp
对象还没创建：拷贝构造
对象已经存在：赋值
```

例如：

```cpp
Date d1(2024, 5, 1);
Date d2;

d2 = d1;       // 赋值，不是拷贝构造
```

这时候 d2 已经存在了，所以这里不是拷贝构造，而是赋值运算符，后面会继续整理。

------

## 四、为什么参数必须用引用？

拷贝构造函数必须使用引用传参。

正确写法：

```cpp
Date(const Date& d)
```

错误写法：

```cpp
Date(Date d)
```

为什么不能传值？

因为如果是传值，调用拷贝构造函数时，需要先把实参拷贝给形参。

例如：

```cpp
Date d2(d1);
```

那么传参时需要先用 d1拷贝构造出形参 d。

但是为了构造形参 d，又要调用拷贝构造函数。

这样就会变成：

```cpp
调用拷贝构造
  -> 传参需要拷贝构造
      -> 传参又需要拷贝构造
          -> 无限递归
```

所以拷贝构造函数的参数必须用引用，避免传参时再次触发拷贝构造。

------

## 五、为什么建议加 const？

拷贝构造函数一般写成：

```cpp
Date(ocnst Date& d)
```

而不是：

```cpp
Date(Date& d)
```

加 const 的原因是：

> 拷贝构造只是读取原对象的数据，不应该修改原对象。

例如：

```cpp
Date(const Date& d) {
    _year = d._year;
    _month = d._month;
    _day = d._day;
}
```

这里 d 时被拷贝的对象，只需要读取它的成员变量。

加上 const 后，可以防止再拷贝构造函数中误修改原对象。

同时，const 引用也可以接收 const 对象和临时对象，适用范围更广。

所以拷贝构造函数的推荐写法是：

```cpp
Date(const Date& d)
```

------

## 六、编译器默认生成的拷贝构造函数

如果我们没有自己写的拷贝构造函数，编译器会自动生成一个默认的拷贝构造函数。

对于日期类这种只有普通成员变量的类：

```cpp
class Date {
    private:
        int _year;
        int _month;
        int _day;
};
```

编译器默认生成的拷贝构造函数一般就够用了。

因为它会把成员变量逐个拷贝：

```cpp
_year 拷贝给 _year
_month 拷贝给 _month
_day 拷贝给 _day
```

这种方式叫**浅拷贝**。

------

## 七、什么是浅拷贝？

浅拷贝就是：

> 直接把类中的成员变量按值拷贝一份。

对于日期类：

```cpp
class Date {
    private:
        int _year;
        int _month;
        int _day;
};
```

浅拷贝没有问题。

因为 _year、_month、_day 都是普通变量，直接复制值即可。

例如：

```cpp
Date d1(2024, 5, 1);
Date d2(d1);
```

拷贝后：

```cpp
d1 有自己的 _year、_month、_day
d2 也有自己的 _year、_month、_day
```

两个对象互不影响。

------

## 八、浅拷贝的问题

浅拷贝有一些类中会出现问题，尤其是类中有指针成员，并且指针指向动态申请的空间。

例如栈类：

```cpp
class Stack {
	public:
		Satck(int capacity = 4) {
			_array = (int*)malloc(sizeof(int) * capacity);
			_top = 0;
			_capacity = capacity;
		}
		
		~Stack() {
			if (_array) {
				free(_array);
				_array = nullptr;
			}
		}
		
	private:
		int* _array;
		int _top;
		int _capacity;
};
```

如果使用默认拷贝构造：

```cpp
Stack s1;
Stack s2(s1);
```

编译器默认拷贝时，会直接把 s1._array 的地址复制给 s2._array。

也就是说：

```cpp
s1._array 和 s2._array 指向同一块动态内存空间
```

这就会出现问题。

------

## 九、为什么会重复释放？

假设 s1 和 s2 的 _array 指向同一块空间。

当对象生命周期结束时：

```cpp
s2 析构，释放 _array 指向的空间
s1 析构，又释放同一块空间
```

同一块动态内存被释放两次，就可能导致程序崩溃。

这种问题的本质是：

> 默认拷贝构造函数只是复制了指针的值，没有复制指针指向的资源。

也就是说，他只复制了地址，没有重新申请一块独立的空间。

------

## 十、析构函数里加 if 能解决问题吗？

很多初学者可能会想：

```cpp
if (_array) {
    free(_array);
    _array = nullptr;
}
```

这样是不是就不会重复释放了？

不能。

因为：

```cpp
_array = nullptr 只会把当前对象置空
不会影响另一个对象里的指针
```

如果 s1._array 和 _s2._array 原来指向同一块空间，s2 析构时把自己的 _array 置空，并不会让 s1._array 也变空。

所以 s1 析构时，仍然可能拿着原来的地址再次释放。

因此，单靠简单的析构函数里的 if 判断不能解决浅拷贝带来的重复释放问题。

------

## 十一、什么是深拷贝？

深拷贝就是：

> 拷贝对象时，不只是复制指针地址，而是重新申请一块空间，把原空间的数据复制过去。

对于栈类，正确思路应该是：

```cpp
Stack(const Stack& s) {
	_array = (int*)malloc(sizeof(int) * s._capacity);
	
	for (int i = 0; i < s._top; i++) {
		_array[i] = s._array[i];
	}
	
	_top = s.top;
	_capacity = s._capacity;
}
```

这样拷贝后：

```cpp
s1 有自己的数组空间
s2 也有自己的数组空间
```

两个对象互不影响，各自析构时释放自己的空间，就不会重复释放。

------

## 十二、什么时候需要自己写拷贝构造？

简单总结：

如果类中只有普通变量，例如：

```cpp
int
double
char
```

一般可以使用编译器默认生成的拷贝构造。

例如日期类：

```cpp
class Date {
    private:
        int _year;
        int _month;
        int _day;
};
```

默认拷贝通常够用。

但是如果类中管理了资源，比如：

```cpp
动态内存
文件指针
网络连接
系统资源
```

就需要写自己的拷贝构造函数，避免浅拷贝带来的资源冲突。

尤其是类中有指针成员时，要重点考虑是否需要深拷贝。

------

## 十三、小结

本篇主要学习了拷贝构造和深浅拷贝。

需要记住：

1. 拷贝构造用于自己已有对象初始化新对象；
2. Date d2(d1) 会调用拷贝构造；
3. Date d2 = d1 也是拷贝构造，不是赋值；
4. 拷贝构造函数参数必须使用引用；
5. 推荐写成 const 类名&；
6. 编译器会默认生成拷贝构造函数；
7. 默认拷贝构造通常是浅拷贝；
8. 日期类这种没有动态资源的类，默认浅拷贝一般够用；
9. 栈类这种有动态资源的类，默认浅拷贝会导致多个对象指向同一块空间；
10. 浅拷贝可能导致析构时重复释放；
11. 析构函数中的 if 判断不能从根本上解决重复释放的问题；
12. 有资源管理的类通常需要自己实现深拷贝。

拷贝构造是类和对象中的重点内容。理解深浅拷贝之后，后面学习赋值运算符重载时，就能明白为什么有些类不能直接依赖编译器默认生成的赋值函数。
