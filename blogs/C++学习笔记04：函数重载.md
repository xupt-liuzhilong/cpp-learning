#  一、什么是函数重载？

函数重载是C++中的一个重要语法。

它允许在同一个作用域中，定义多个同名函数，但这些函数的**参数列表必须不同**。

参数列表不同主要包含三种情况：

1. 参数个数不同；
2. 参数类型不同；
3. 参数顺序不同。

简单来说就是：

> 函数名字可以相同，但参数必须有区别。

---

# 二、参数个数不同

```cpp
#include <iostream>
using namespace std;

void Func() {
	cout << "Func()" << endl;
}

void Func(int a) {
	cout << "Func(int)" << endl;
}

int main () {
	Func();
	Func(10);
	
	return 0;
}
```

这两个函数名字都叫Func，但是一个没有参数，一个有一个int参数，所以可以构成函数重载。

# 三、函数类型不同


# 四、函数顺序不同


# 五、返回值类型不同不能构成重载

这是函数重载里最容易出错的地方，

下面这种写法是错误的：

```cpp
int Func(int a) {
	return a;
} 

double Func(int a) {
	return a;
}
```

![](data:image/gif;base64,R0lGODlhAQABAPABAP///wAAACH5BAEKAAAALAAAAAABAAEAAAICRAEAOw== "点击并拖拽以移动")

虽然这两个函数的返回值不同，但他们的参数列表完全相同，所有能构成函数重载。

原因是函数调用时如果这样写：

```cpp
Func(10);
```

![](data:image/gif;base64,R0lGODlhAQABAPABAP///wAAACH5BAEKAAAALAAAAAABAAEAAAICRAEAOw== "点击并拖拽以移动")

编译器无法判断你是想调用int的版本， 还是法案会都变了的版本。

所以要记住：

> 函数重载只看参数列表，不看返回值。

# 六、为什么C++支持函数重载

C++支持函数重载，是因为编译器会对函数名进行处理。

在底层，C++编译器会把函数名和参数类型组合起来，生成一个新的名字。

例如：

```cpp
int Add(int a, int b);
double Add(double, double b);
```

![](data:image/gif;base64,R0lGODlhAQABAPABAP///wAAACH5BAEKAAAALAAAAAABAAEAAAICRAEAOw== "点击并拖拽以移动")

在底层可能会被处理成类似：

```cpp
Add_int_int
Add_double_double
```

![](data:image/gif;base64,R0lGODlhAQABAPABAP///wAAACH5BAEKAAAALAAAAAABAAEAAAICRAEAOw== "点击并拖拽以移动")

所以虽然代码里面函数名都叫Add，但是编译器底层能区分它们。

这是C++支持函数承载，而C语言不支持函数重载的原因之一。

# 七、小结

本篇主要学习了C++中的函数重载。

需要记住：

1. 函数重载允许多个同名函数存在；
2. 重载函数必须在同一个作用域中；
3. 参数个数不同可以构成重载；
4. 参数类型不同可以构成重载；
5. 参数顺序不同可以构成重载；
6. 仅返回值不同不能构成重载；
7. C++通过函数名修饰规则支持重载函数。

函数重载可以让一个功能相似的函数使用同一个名字，提高代码可读性。比如加法函数可以统一叫Add，编译器会根据传入的参数自动选择合适的版本。
