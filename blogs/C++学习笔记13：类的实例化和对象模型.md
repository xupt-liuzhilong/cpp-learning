# C++学习笔记13：类的实例化和对象模型

## 一、什么是类的实例化？

在C++中，类只是一个自定义类型，本身不会直接占用对象空间。

比如定义一个 Date 类：

```cpp
class Date {
    public:
        int _year;
        int _month;
        int _day;
};
```


这里只是告诉编译器：Date 这个类型里面有哪些成员。

如果真的想使用它，需要通过类创建对象：

```cpp
Date d1;
Date d2;
```


这个过程叫**类的实例化**。

简单理解：

> 类是一个设计图，对象是根据设计图创建出来的实体。

比如 Date 是一个日期类，d1、d2 就是具体的日期对象。

------

## 二、类和对象的关系

类和对象的关系可以类比成：

```cpp
类：图纸
对象：根据图纸造出来的房子
```


同一张图纸可以建很多栋房子。

同一个类也可以创建很多个对象。

例如：

```cpp
Date d1;
Date d2;
Date d3;
```


这里d1、d2、d3 都是 Date 类型的对象。

他们的成员变量是独立的：

```cpp
d1._year = 2024;
d2._year = 2025;
d3._year = 2026;
```


修改 d1 成员变量，不会影响 d2 和 d3。

------

## 三、对象里面存了什么？

类里面可以有成员变量和成员函数。

例如：

```cpp
class Date {
	public:
		void Print() {
			cout << _year << " " << _month << " " << _day << endl; 
		}
		
	private:
		int _year;
		int _month;
		int _day;
};
```

那么问题来了：

> 对象里面到底存的是成员变量，还是成员函数一起存呢？

答案是：

> 对象中只存成员变量，成员函数不存放在对象里面。

也就是说，Date 对象真正存储的是：

```cpp
int _year;
int _month;
int _day;
```


而 Print 函数不会在每一个对象里都存一份。

------

## 四、为什么成员函数不存进对象？

假设有三个对象？

```cpp
Date d1;
Date d2;
Date d3;
```


他们的年月日可能不同，所以成员变量必须每个对象单独保存。

但是成员函数和代码是一样的。

比如 Print() 函数，不管是 d1.Print()、d2.Print() 还是 d3.Print()，执行的都是同一份函数代码。

如果每一个对象都保存一份函数代码，就会浪费空间。

所以C++的做法是：

```cpp
成员变量：每个对象单独保存
成员函数：所有对象共享一份
```


成员函数一般放在公共代码区，对象只保存自己的成员变量。

------

## 五、sizeof 对象的结果

因为队形只存成员变量，所以 sizeof 计算对象大小时，主要看成员变量。

例如：

```cpp
#include <iostream>
using namespace std;

class Date {
	public:
		void Print() {
			cout << _year << " " << _month << " " << _day << endl; 
		}
		
	private:
		int _year;
		int _month;
		int _day;
}; 

int main() {
	Date d;
	
	cout << sizeof(d) << endl;
	cout << sizeof(Date) << endl;
	
	return 0;
}
```

一般情况下，输入结果是：

```cpp
12
12
```


因为 Date 类中有3个 int 成员变量，每个 int 通常占4个字节，所以对象的大小是12字节。

注意：

```cpp
sizeof(d);
sizeof(Date);
```


这两个结果都是一样的，都是计算 Date 对象所占空间大小。

------

## 六、成员函数不影响对象大小

即使类中有成员函数，也不会增加对象大小。

例如：

```cpp
class A {
	public:
		void Func1() {
			cout << "Func1" << endl;
		}
		
		void Func2() {
			cout << "Func2" << endl;
		}
		
	private:
		int _a;
};
```

这个类的对象大小主要由 _a 决定。

虽然类中有 Func1 和 Func2，但他们不窜访在对象里面，所以不会让每个对象都变大。

------

## 七、空类的大小为什么是 1？

如果一个类中没有成员变量：

```cpp
class Empty {

};
```


那么它的大小是多少呢？

```cpp
cout << sizeof(Empty) << endl;
```


结果不是 0，而是：

```cpp
1
```


原因是：

> 空类也需要有唯一的地址。

如果空类的大小是 0，那么定义多个对象时，他们可能无法区分地址。

例如：

```cpp
Empty e1;
Empty e2;
```


即使类里面什么都没有，e1 和 e2 也应该是两个不同的对象。

所以编译器会给空类对象分配 1 字节，用来占位和区分不同。

------

## 八、小结

本篇主要学习了类的实例化和对象模型。

需要记住：

1. 类是定义类型，对象是类实例化出来的实体；
2. 类可以理解为设计图，对象可以理解成实体；
3. 同一个类可以创建多个对象；
4. 不同对象的成员变量是独立的；
5. 对象主要存放成员变量；
6. 成员函数存放在公共代码区，所个对象同时共用一份代码函数；
7. sizeof(对象) 和 sizeof(类名) 都可以计算对象大小；
8. 空类的大小是1字节，用来区分不同对象的地址。

类的实例化和对象模型是理解C++类和对象的基础。后面学习 this 指针时，就能进一步理解：成员函数虽然是共享的，但为什么知道自己草所得是哪一个对象。
